# Memory / Knowledge Capture Policy

Use when preserving lessons from a coding session.

## Generalization rule

Before converting project-specific lessons into this generic skill, remove or generalize all private names and business context. If a lesson cannot be generalized safely, do not include it in this public/generic skill.

## Remember as generic knowledge

- Stable engineering rules.
- Reusable pitfalls.
- Framework/library quirks.
- Project-neutral verification commands.
- Architecture patterns after removing private names.

## Do not remember in public/generic skill

- Company names, internal service names, app keys, URLs, table names, proprietary business processes.
- Temporary branch context.
- Credentials or secrets.
- Incident details that identify a private system.
- Customer/user data, production record identifiers, or private operational timelines.

## End-of-session checklist

- Pitfalls → generic memory or reference file.
- Stable rules → skill/reference guidance.
- Temporary context → do not persist unless needed for the current repo.
- Private details → omit or generalize.
